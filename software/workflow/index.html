Verschil tussen Hugo en een klassieke Wordpress website:

Om eerlijk te zijn… Ik heb Wordpress nog niet echt intensief gebruikt, maar wel al om eens te bekijken wat het juist kan. Ik ga dus mijn best doen om het verschil zo goed mogelijk te verwoorden.

Wanneer je bij een Hugo website iets wil aanpassen aan de layout moet je code bewerken in de layout map en in de static map. Voor de content maak je verschillende markdown -of yamlbestandjes aan in de content-map. In de layout map staan de verschillende componenten (partials) en in de static kan je vooral CSS en javascript terugvinden.

Bij een Wordpress website kan je alles via de browser bewerken. Je klikt gewoon op het bericht dat je wil bewerken en een hele hoop toepassingen staan ter beschikking. Bij Wordpress is het niet zo belangrijk om op de hoogte te zijn van de code, maar bij Hugo moet je er wel van op de hoogte zijn. Uiteraard is het wel een meerwaarde bij Wordpress als je iets van code afweet. Het biedt je zo meer mogelijkheden.

Directory-structuur website:

  • In de content-map van mijn website staan alle markdown bestandjes waarin ik de tekst aan kan passen van mijn posts. Ik heb voor elke taak een nieuw bestandje in de bijhorende map. Bijvoorbeeld de FAQ in het Frans staat in de map ‘Frans’. Zo kan ik elke taak netjes gestructureerd terugvinden.

  • In de stage-map zal ik alle stageverslagen plaatsen wanneer de stageperiode begint. Deze hangt samen met een list waardoor alles mooi onder elkaar komt te staan op datum.

  • In de layout-map staan nog eens de mappen zoals ze in de content-map staan, maar dan zonder de markdownbestandjes. Hier staan single.html documenten in die de opmaak weergeven voor de pagina die u te zien zal krijgen.

  • Bij partials staan de bepaalde sections van mijn site. Neem nu about.html als voorbeeld. Dit verwijst naar de info op de hoofdpagina waar u de link ‘Ga naar GIP’ en ‘Ga naar Stage’ terug kan vinden.

  • De Sidebar, footer en head_hyde staan voor de layout van de GIP pagina.

  • De sidebar2, footer en head_hyde2 staan voor de layout van de Stage pagina.

  • In de static-map kan je de CSS en javascript bestanden terugvinden.

Ik heb 2 thema’s samengevoegd, maar heb wel heel veel aangepast. Bijna niets is nog in zijn standaard vorm. Op de ene plaats is er gewoon een kleurtje aangepast en op de andere plaats is eigenlijk alles anders.

Moeilijkheden:

Ik heb de eerste lessen heel wat problemen ondervonden bij de werking van Hugo. Ik begreep absoluut niet waar ik wat kon vinden. Na heel wat uit te proberen begon het te beteren en plots ging het heel vlot. Mijn website kreeg vorm en alles leek te lukken. Uiteindelijk heb ik 2 thema’s samengevoegd. Bij het publiceren van de website heb ik ook heel wat problemen gehad. Github wilde mijn website niet weergeven zoals het zou moeten, maar uiteindelijk is het dan toch gelukt.

Heb ik iets bijgeleerd en zou ik Hugo nog gebruiken in de toekomst?

Ik heb heel wat bijgeleerd tijdens het maken van mijn website. Ik leerde heel wat nieuwe code bij en ook een nieuwe codetaal namelijk GO. Ik heb meer inzicht gekregen in de code en de structuur ervan.

Zou ik Hugo ooit nog eens gebruiken? Ik denk het wel! Het is een leuke manier om website’s te maken, het is leerrijk en je kan er mooie eindresultaten mee bereiken.
